## 2. Git Flow / Flujo de trabajo

1. **main**: Rama principal con la versión estable.  
2. **develop**: Rama de desarrollo donde se integran nuevas funcionalidades antes de `main`.  
3. **feature/nombre-funcionalidad**: Ramas temporales para desarrollar nuevas funciones.  
4. **bugfix/nombre-error**: Ramas temporales para corregir errores en `develop` o `main`.  
5. **release/nombre-version**: Preparación de versiones antes de fusionarlas con `main`.  
6. **merge**: Se integran `feature` a `develop`, `develop` a `release` y `release` a `main`.  
7. **tags**: Se crean tags en `main` para marcar versiones oficiales.
